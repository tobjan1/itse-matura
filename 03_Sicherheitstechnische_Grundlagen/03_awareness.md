# Security Awareness

Security Awareness bezeichnet das Sicherheitsbewusstsein von Benutzern und Mitarbeitern. Technische Schutzmaßnahmen alleine reichen nicht aus — der Mensch ist oft die schwächste Stelle in der Sicherheitskette (Human Factor).

## Warum ist Awareness wichtig?

Viele erfolgreiche Angriffe setzen nicht an technischen Schwachstellen an, sondern nutzen menschliches Verhalten aus:

- Phishing-E-Mails werden angeklickt
- Passwörter werden weitergegeben (Social Engineering)
- USB-Sticks unbekannter Herkunft werden eingesteckt
- Vertrauliche Informationen werden am Telefon preisgegeben (Vishing)

## Angriffsformen auf den Menschen (Social Engineering)

| Angriff | Beschreibung |
|---------|-------------|
| **Phishing** | Gefälschte E-Mails mit Links zu Fake-Seiten oder Anhängen mit Malware |
| **Spear Phishing** | Gezieltes Phishing auf bestimmte Personen/Organisationen — personalisiert |
| **Vishing** | Voice Phishing — Angriff per Telefon (z. B. falscher IT-Support) |
| **Smishing** | Phishing per SMS |
| **Pretexting** | Angreifer erfindet einen Vorwand ("Ich bin von der IT, ich brauche Ihr Passwort") |
| **Baiting** | Köder verwenden (infizierter USB-Stick auf Parkplatz legen) |
| **Tailgating** | Physisch in gesichertes Gebäude hinter berechtigter Person hereinschleichen |

## Awareness-Maßnahmen in Organisationen

- **Schulungen**: Regelmäßige Trainings zu aktuellen Bedrohungen
- **Phishing-Simulationen**: Kontrollierte Test-Angriffe, um Bewusstsein zu schärfen
- **Klare Richtlinien**: Passwortrichtlinien, Umgang mit sensiblen Daten, Meldepflichten
- **Meldewege**: Mitarbeiter wissen, wen sie bei Verdacht kontaktieren sollen
- **Clean-Desk-Policy**: Keine sensiblen Unterlagen offen liegen lassen
- **Passwortsicherheit**: Passwort-Manager einsetzen, keine Passwörter weitergeben

## Sicherheitskultur

Awareness ist keine einmalige Maßnahme, sondern ein kontinuierlicher Prozess. Mitarbeiter müssen ermutigt werden, Verdächtiges zu melden — ohne Angst vor Konsequenzen.
